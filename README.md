# KIOSK_PAGE
키오스크 홈페이지 - 이미지/영상 슬라이드쇼 시스템

GitHub Pages를 통해 배포되는 키오스크용 이미지 및 영상 포스터 슬라이드쇼입니다.

## 📁 파일 업로드 규칙

### upload 폴더에 파일 업로드 방법

1. **파일명 규칙**
   - 파일명은 **2자리 숫자**로 시작해야 합니다
   - 형식: `01.jpg`, `02.png`, `03.mp4` 등
   - 예시: `01.jpg`, `02.png`, `03.mp4`, `04.jpg`, `05.webm`

2. **지원하는 확장자**

   **이미지:**
   - `.jpg` / `.JPG`
   - `.jpeg` / `.JPEG`
   - `.png` / `.PNG`
   - `.gif` / `.GIF`
   - `.webp` / `.WEBP`

   **영상:**
   - `.mp4` / `.MP4`
   - `.webm` / `.WEBM`
   - `.ogg` / `.OGG`

3. **업로드 순서**
   - 파일은 파일명의 숫자 순서대로 표시됩니다
   - `01.jpg` → `02.jpg` → `03.mp4` → `04.jpg` 순서로 슬라이드됩니다

4. **슬라이드 동작 방식**
   - **이미지**: 12초 간격으로 자동 전환됩니다
   - **영상**: 영상이 끝까지 재생된 후 다음 슬라이드로 자동 전환됩니다
   - 모든 슬라이드는 fade in/out 효과로 전환됩니다
   - 마지막 슬라이드 후 첫 슬라이드로 무한 반복됩니다

5. **주의사항**
   - 파일은 반드시 `upload` 폴더 안에 있어야 합니다
   - 파일명이 `01`, `02` 형식이 아니면 인식되지 않습니다
   - 영상은 자동 재생되며 음소거(muted) 상태로 재생됩니다

### 예시
```
upload/
  ├── 01.jpg
  ├── 02.jpg
  ├── 03.mp4
  ├── 04.jpg
  └── 05.png
```

## ⚙️ 시스템 특징

### 자동 슬라이드 동작

- **이미지 슬라이드**: 12초 간격으로 자동 전환
- **영상 슬라이드**: 영상 재생 완료 후 자동 전환
- **전환 효과**: Fade in/out (1초 전환 시간)
- **무한 반복**: 마지막 슬라이드 후 첫 슬라이드로 자동 반복
- **터치 비활성화**: 키오스크 환경에 맞게 터치/드래그 비활성화

### 화면 구성

- **전체 화면**: 100vw × 100vh (키오스크 화면에 최적화)
- **이미지/영상 표시**: `object-fit: contain` (비율 유지하며 전체 표시)
- **배경색**: 검은색 (#000)

### 설정 변경 방법

현재 설정을 변경하려면 `index.html` 파일의 JavaScript 코드를 직접 수정해야 합니다:

- **이미지 전환 시간 변경**: `swiperInstance.params.autoplay.delay = 12000;` 부분의 숫자 변경 (밀리초 단위)
- **전환 속도 변경**: `speed: 1000` 부분의 숫자 변경 (밀리초 단위)
- **전환 효과 변경**: `effect: 'fade'` 부분을 `'slide'`로 변경 가능

## 🚀 GitHub Pages 배포 방법

1. **저장소 생성 및 파일 업로드**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/사용자명/저장소명.git
   git push -u origin main
   ```

2. **GitHub Pages 활성화**
   - GitHub 저장소 페이지로 이동
   - Settings → Pages 메뉴 선택
   - Source에서 `main` 브랜치 선택
   - Save 클릭

3. **배포 확인**
   - 배포 완료 후 `https://사용자명.github.io/저장소명/` 주소로 접속
   - 이미지와 설정이 정상적으로 로드되는지 확인

## 📝 주의사항

- 파일은 반드시 `upload` 폴더 안에 있어야 합니다
- 파일명은 2자리 숫자로 시작해야 하며, 앞에 0을 붙여야 합니다 (예: `01`, `02`, `10`)
- 영상 파일은 브라우저가 지원하는 코덱 형식이어야 합니다 (권장: MP4 H.264)
- GitHub Pages는 정적 파일만 호스팅하므로 서버 측 처리가 필요 없습니다
- 키오스크 환경에서는 브라우저를 전체 화면 모드로 실행하는 것을 권장합니다
